# SkyJSON

**A lightweight dashboard for real-time aircraft data.**

SkyJSON is a self-hosted Linux web application that reads aircraft data from an `aircraft.json` file and displays it in a clean, searchable dashboard.

## Highlights

- Web-based first-run installation wizard
- Configure the path to `aircraft.json` from the browser
- Optional password protection for the configuration panel
- Dashboard with search, stats and automatic refresh
- Built-in **Update from GitHub** button in the configuration panel
- Choice between **standalone mode** and **lighttpd reverse proxy mode**
- Ready-to-use **lighttpd** and **systemd** example files in `deploy/`

## Supported sources

SkyJSON works with `aircraft.json` files generated by tools such as:

- dump1090
- dump1090-fa
- readsb
- tar1090

Typical paths include:

```bash
/run/dump1090-fa/aircraft.json
/var/run/dump1090-fa/aircraft.json
/run/readsb/aircraft.json
```

## Quick start

Clone the repository:

```bash
git clone https://github.com/PatrickS86/skyjson.git
cd skyjson
```

Create a virtual environment and install dependencies:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Run SkyJSON:

```bash
python3 app.py
```

Open:

```text
http://localhost:8000
```

On first launch, SkyJSON opens the web installer. There you can:

1. Set the application title
2. Enter the path to `aircraft.json`
3. Choose the refresh interval and dashboard size
4. Choose whether the configuration panel should be protected with a username and password
5. Choose whether SkyJSON will run standalone or behind lighttpd

## Running SkyJSON with lighttpd

SkyJSON can run perfectly on systems where no webserver is installed yet. A common setup is:

- `lighttpd` handles incoming HTTP or HTTPS traffic
- `gunicorn` runs SkyJSON on `127.0.0.1:8000`
- `lighttpd` proxies requests to Gunicorn

Install the base packages:

```bash
sudo apt update
sudo apt install lighttpd python3 python3-venv python3-pip git
```

Copy the included service and webserver examples:

```bash
sudo cp deploy/systemd/skyjson.service /etc/systemd/system/skyjson.service
sudo cp deploy/lighttpd/skyjson.conf /etc/lighttpd/conf-available/99-skyjson.conf
sudo lighty-enable-mod proxy
sudo lighty-enable-mod setenv
sudo lighty-enable-mod accesslog
sudo lighty-enable-mod expire
sudo lighty-enable-mod alias
sudo lighty-enable-mod rewrite
sudo lighty-enable-mod access
sudo lighty-enable-mod status
sudo lighty-enable-mod 99-skyjson
sudo systemctl daemon-reload
sudo systemctl enable --now skyjson
sudo systemctl restart lighttpd
```

Then edit the host name in `skyjson.conf` and adjust paths in the service file to match your installation directory.

## Standalone mode

Standalone mode is useful for quick testing or internal-only deployments. Use the included file:

- `deploy/systemd/skyjson-standalone.service`

That mode starts the built-in Python server directly on port `8000`.

## Updating from GitHub

The configuration page includes an **Update from GitHub** button.

How it works:

- SkyJSON checks whether the app was installed from a Git repository
- It runs `git fetch origin` to compare your local version with the remote version
- When an update is available, you can install it from the browser
- The update action uses `git pull --ff-only`

Requirements for web updates:

- `git` must be installed on the server
- the application directory must be a cloned Git repository
- the user running SkyJSON must have write access to the repository folder
- after updating, restart the app service when needed

## Configuration security

You can keep the dashboard public and only protect `/config`.

If configuration protection is enabled:

- the dashboard remains accessible
- the configuration page requires a username and password
- passwords are stored as hashed values in the local SQLite database

## Notes

- For public-facing deployments, lighttpd + gunicorn is recommended over the built-in Flask server.
- The web installer only stores the chosen deployment mode in SkyJSON; it does not modify system files automatically.
- The included deployment files are templates and may need small path or hostname changes.
